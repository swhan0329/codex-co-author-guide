English guide

Add Codex to your Git commit history without changing your normal workflow.

Use the sample-codex-coauthor pattern to automatically append Co-authored-by: codex <codex@openai.com> to your commits.

Real example Markdown source
Setup model
Once per local clone

Codex install is user-level. The co-author hook is repo-level.

What stays manual
Push, PR, merge

The hook only changes the commit trailer, not your GitHub workflow.

Basic flow

  1. Clone the target repository
  2. Run bash scripts/setup-codex-attribution.sh once
  3. Work with Codex app or CLI
  4. Run tests
  5. Commit normally
  6. Verify the commit body
  7. Push and open a pull request
bash scripts/setup-codex-attribution.sh
git add .
git commit -m "Fix small bug"
git show -s --format=%B HEAD

Why setup must be repeated in a new clone

Because the setup script writes to local Git config in .git/config. The files can live in the repository, but activation still happens per clone.

The original sample repo supports both scripts and Codex skills

Script path
bash scripts/setup-codex-attribution.sh
Skill path
  • $install-codex-coauthor-hook
  • $add-ai-coauthor
  • $show-ai-attribution-status

For beginners, the script path is the cleanest explanation. For a full write-up of the original sample repo, it is better to mention that both paths exist.
