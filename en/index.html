Practical English guide Codex co-author trailer automation

Add Codex to your Git commit history without changing your normal workflow.

Use the sample-codex-coauthor pattern to automatically append Co-authored-by: codex <codex@openai.com> to your commits.

Read the real example Markdown source

When to use it

Use it if
  • you work with Codex app or CLI
  • you still create commits yourself
  • you want GitHub to show Codex as a co-author
Skip it if
  • you do not care about the trailer
  • your normal commit flow is already enough

Basic flow

1

Clone the repository

Start with a normal local clone.

2

Run setup once

bash scripts/setup-codex-attribution.sh

3

Work with Codex

Use Codex app or CLI inside the repository.

4

Commit normally

The hook appends the trailer automatically.

bash scripts/setup-codex-attribution.sh
git add .
git commit -m "Fix small bug"
git show -s --format=%B HEAD

Why setup must be repeated in a new clone

Because the script writes local Git configuration into .git/config. Files can live in the repository, but activation still happens per clone.

The original sample repo also supports a skill path

The original sample repo supports both a direct script path and a repo-local Codex skill path.

Script path
bash scripts/setup-codex-attribution.sh
Skill path
  • $install-codex-coauthor-hook
  • $add-ai-coauthor
  • $show-ai-attribution-status
