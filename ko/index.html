한국어 실전 가이드 Codex co-author trailer 자동화

Codex를 GitHub co-author처럼 남기는 가장 쉬운 방법

이 문서는 sample-codex-coauthor를 실제 레포에 적용해서 Co-authored-by: codex <codex@openai.com>를 자동으로 붙이는 흐름을 설명합니다.

vehicle_speed_estimation 예시 Markdown source

언제 쓰면 되나

쓰면 좋은 경우
  • Codex app 또는 CLI로 수정할 예정
  • 직접 git commit 할 예정
  • GitHub에서 Codex co-author 표시를 남기고 싶음
굳이 안 써도 되는 경우
  • Codex는 쓰지만 co-author 표시가 필요 없음
  • 그냥 일반 커밋 흐름이면 충분함

기본 흐름

1

레포를 clone 한다

작업할 레포를 먼저 로컬로 받습니다.

2

setup 스크립트를 한 번 실행한다

bash scripts/setup-codex-attribution.sh

3

Codex로 수정한다

Codex app 또는 CLI로 작업합니다.

4

테스트 후 평소처럼 커밋한다

co-author trailer는 훅이 자동으로 붙입니다.

bash scripts/setup-codex-attribution.sh
git add .
git commit -m "Fix small bug"
git show -s --format=%B HEAD

왜 새 clone에서는 다시 해야 하나

setup 스크립트는 레포 파일이 아니라 로컬 Git 설정인 .git/config를 바꿉니다.

정리: 같은 clone에서는 한 번만 하면 되고, 새 clone이나 다른 컴퓨터에서는 다시 실행해야 합니다.

원본 sample repo는 skill로도 쓸 수 있다

원본 repo는 스크립트 방식뿐 아니라 repo-local Codex skills도 제공합니다.

스크립트 방식
bash scripts/setup-codex-attribution.sh
skill 방식
  • $install-codex-coauthor-hook
  • $add-ai-coauthor
  • $show-ai-attribution-status

초심자에게는 스크립트 방식이 가장 쉽습니다. 하지만 원본 sample repo를 소개할 때는 skill 경로도 함께 설명해주는 것이 더 정확합니다.
