Codex를 GitHub co-author처럼 남기는 가장 쉬운 방법

이 문서는 sample-codex-coauthor를 실제 레포에 적용해서 Co-authored-by: codex <codex@openai.com>를 자동으로 붙이는 흐름을 설명합니다.

한국어 예시 보기 Markdown source
핵심
레포 clone마다 한 번

Codex 설치가 아니라, 특정 레포의 로컬 Git 설정을 켜는 단계입니다.

자동화 범위
commit trailer 추가

push, PR, merge는 여전히 평소 GitHub 흐름대로 진행합니다.

언제 쓰면 되나

쓰면 좋은 경우

  • Codex app 또는 CLI로 수정할 예정
  • 직접 git commit 할 예정
  • GitHub에서 Codex co-author 표시를 남기고 싶음

굳이 안 써도 되는 경우

  • Codex는 쓰지만 co-author 표시가 필요 없음
  • 그냥 일반 커밋 흐름이면 충분함

자동으로 붙는 문장

Co-authored-by: codex <codex@openai.com>

기본 흐름

  1. 레포를 clone 한다
  2. bash scripts/setup-codex-attribution.sh를 한 번 실행한다
  3. Codex app 또는 CLI로 작업한다
  4. 테스트를 돌린다
  5. 평소처럼 git commit 한다
  6. 커밋 본문에 trailer가 붙었는지 확인한다
  7. git push 후 PR을 연다
bash scripts/setup-codex-attribution.sh
git add .
git commit -m "Fix small bug"
git show -s --format=%B HEAD

왜 새 clone에서는 다시 해야 하나

setup 스크립트는 레포 파일이 아니라 로컬 Git 설정인 .git/config를 바꿉니다.

다시 안 해도 되는 경우
  • 같은 컴퓨터
  • 같은 clone
다시 해야 하는 경우
  • 새 컴퓨터
  • 새 clone

원본 sample repo는 skill로도 쓸 수 있다

원본 repo는 스크립트 방식뿐 아니라 repo-local Codex skills도 제공합니다.

스크립트 방식
bash scripts/setup-codex-attribution.sh
skill 방식
  • $install-codex-coauthor-hook
  • $add-ai-coauthor
  • $show-ai-attribution-status

초심자에게는 스크립트 방식이 제일 쉽습니다. 하지만 원본 sample repo를 소개할 때는 “스크립트로도 되고, Codex skill로도 된다”까지 같이 적어주는 게 가장 정확합니다.

더 자세한 적용 흐름은 vehicle_speed_estimation 한국어 예시 에 정리되어 있습니다.
