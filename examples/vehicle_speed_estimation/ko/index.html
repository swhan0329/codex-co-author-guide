실제 레포 예시 swhan0329/vehicle_speed_estimation

vehicle_speed_estimation을 clone한 뒤 실제로 무엇을 하면 되나

이 페이지는 비교 설명보다 실제 실행 순서에 집중합니다. vehicle_speed_estimation을 clone한 사람이 Codex co-author trailer가 자동으로 붙게 하려면 어떤 명령을 치면 되는지 단계별로 적었습니다.

목표: Co-authored-by: codex <codex@openai.com>가 커밋에 자동 추가되게 만들기

1. 레포를 clone 한다

git clone https://github.com/swhan0329/vehicle_speed_estimation.git
cd vehicle_speed_estimation

2. co-author 자동 추가를 켠다

이 레포에는 이미 필요한 파일이 들어 있으므로, 아래만 실행하면 됩니다. 

bash scripts/setup-codex-attribution.sh

이 단계는 같은 clone에서는 한 번만 하면 됩니다. 새 clone이면 다시 해야 합니다.

3. Codex로 레포를 연다

  • Codex app: vehicle_speed_estimation 폴더를 연다
  • Codex CLI: 레포 폴더에서 codex를 실행한다
이 레포에서 작은 개선점 하나를 찾아 수정하고 테스트까지 돌려줘.

4. 테스트를 돌린다

이 레포는 둘 다 됩니다.

pytest -q

또는

python -m unittest discover -s tests

5. 평소처럼 commit 한다

git add .
git commit -m "Fix small issue in vehicle_speed_estimation"

이때 Codex trailer를 직접 적을 필요는 없습니다. 훅이 자동으로 붙입니다.

6. 실제로 붙었는지 확인한다

git show -s --format=%B HEAD

정상이라면 이렇게 보입니다.

Fix small issue in vehicle_speed_estimation

Co-authored-by: codex <codex@openai.com>

7. 원격으로 push 한다

git push origin <브랜치명>

그 다음은 평소처럼 PR을 열고 GitHub에서 merge 하면 됩니다.
