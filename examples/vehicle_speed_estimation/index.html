What to do after cloning vehicle_speed_estimation

This page is intentionally procedural. It explains the exact flow for a person who cloned vehicle_speed_estimation and wants future commits in that clone to automatically include the Codex co-author trailer.

Target trailer: Co-authored-by: codex <codex@openai.com>

1. Clone the repository

git clone https://github.com/swhan0329/vehicle_speed_estimation.git
cd vehicle_speed_estimation

2. Enable the co-author hook

The repository already contains the required files, so run the setup script once in this local clone. 

bash scripts/setup-codex-attribution.sh

This writes local Git config and must be repeated in a new clone.

3. Open the repository with Codex

  • Codex app: open the vehicle_speed_estimation folder
  • Codex CLI: run codex inside the repository
Find a small improvement in this repository, fix it, and run the tests.

4. Run tests

Either command works in this repository.

pytest -q

or

python -m unittest discover -s tests

5. Commit normally

git add .
git commit -m "Fix small issue in vehicle_speed_estimation"

You do not need to type the Codex trailer yourself. The hook appends it automatically.

6. Verify the commit body

git show -s --format=%B HEAD

Expected result:

Fix small issue in vehicle_speed_estimation

Co-authored-by: codex <codex@openai.com>

7. Push the branch

git push origin <branch-name>

After that, open a PR and merge on GitHub as usual.
