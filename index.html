Two valid ways to use the original sample repo

The original sample repository supports both a direct script path and a more agent-native Codex skill path.

Script path

Run the setup shell script directly

bash scripts/setup-codex-attribution.sh

Best default for beginners because it is explicit and easy to teach in a README or blog post.

Skill path

Ask Codex to use repo-local skills

  • $install-codex-coauthor-hook
  • $add-ai-coauthor
  • $show-ai-attribution-status

Useful when you want the workflow to feel native inside Codex.

Practical path

Keep a production repo minimal

  • .githooks/prepare-commit-msg
  • .gitmessage
  • scripts/setup-codex-attribution.sh

This is the setup used in the real example repository.
